Penetration Tester

Inspiration

My inspiration behind starting this project was that I wanted to make an anti virus application ,but I soon
found out that it would take alot research to carry out. Then it hit me that I could make an application
that compromises systems instead of defending them. I found a book that I would base my research on and take
me down the necessary paths. That book being; 'The basics of hacking and penetration testing' by

Overview

As my resarch highlighted in most hacking there are four main steps:

  1. Reconnaissance
  2. Scanning
  3. Exploitation
  4. Post-Exploitation Control*
but my application only features the first 3 due to restrictions placed on me by my school.
Reconnaisance

The reconnaissance pahse is the part of the hack where the hackler finds things about their targets.
This includes finding what kind of servers are on the network. The IP of the clients on the network.

Scanning

Information is passed across and within networks and clinets on those network by sending packets of data
from the ports of clients to other clients or any gateway in the network. Scanning a clinet on a network includes checking each port to see what protocols are running on those ports
Common protocols include TCP, UDP and much more*
Upon finding this we can accurately find vulnerabilities which we can use to exploit the system in th enext step

Exploitation

This step uses the vulnerabilities we found within. The scanning Phase to exploit the system. It is important to note here that I didnt actually hack anyone. I created a virtual network using VMware
Then I made a network with 1 client and one server. The server was equipt with active directory to make it more realistic

Execution

To make this project possible I had to use some external modules such as a python module made after nmap as well as
a module made after metaslpoit.
Nmap was a module that I used to scanning the client I wanted to exploit. Nmap ran a scan and provided very
good results that I had to make 'pretty' to present back to the user.

The metaslpoit framework was very exciting to use. It gets updated very frequently with new exploit and it was
very fun in the end to see each step or phase line up with each other.

Because metasploit was a framework that was much easier to use on linux machines I have to make another
viurtual machine to host the metaploit framework. Even though it was inconvenient it provided a unique oppourtunity
To learn to use the unix machines and get used to operating using the command line
